# CLAUDE.md

**Start here before doing any work on this project.**

1. Read **`work/index.md`** — the living overview of where the project currently
   stands: status by phase, file map, known issues, and how to run things.
2. Check the **"Last updated"** date at the top of `work/index.md`. If it looks
   stale (older than the most recent commits, or the code clearly diverged),
   trust the code and update the index as part of your change.

## How `work/` is organized
- `work/index.md` — current state of the whole project (read first).
- `work/` may also hold detailed design notes for specific features in their own
  markdown files or subfolders. Link new ones from `work/index.md`.

## Maintenance rule
When you finish a meaningful change, **update `work/index.md`**: bump its
"Last updated" date, adjust the status, and add/repoint any links. Keep it short
and current — it is the first thing the next session reads.
